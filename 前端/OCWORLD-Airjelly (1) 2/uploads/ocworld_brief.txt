01  项目定义 · Project Definition


1.1 我们不是另一个聊天 Agent

市场上已经有几千万人在跟 AI 角色聊天——豆包、deepseek、secondMe。但聊完之后呢？


角色还是不懂你——它没有持久记忆，每次都从零开始


它不会按照你的风格做事——只能在对话框里转圈


它更不会跟你一起成长——你今天问什么明天问什么，它都一样

所以我们要做的不是又一个聊天 Agent。我们要做的是——

💡 人生游戏引擎

你的 OC 是跟你一起闯关的搭档，不是工具。它住在你的桌面上，看见你的真实生活；当你准备好，它会带着你的灵魂、你的技能、你的偏好，进入 Agent 世界——替你接任务、跟其他 OC 协作，把"完成任务"变成"打怪升级"。

02  市场洞察与用户痛点 · Market & Pain Points


2.1 三个真实痛点


① 情感孤岛——"AI 不懂我"

当前主流 AI 角色产品（secondMe、星野等）共同的硬伤：


对话即遗忘，没有跨会话的持久记忆


不感知用户真实生活，只能在虚构对话框里反复绕圈


不会成长，关系永远停在 Day 1

用户长期使用后会出现"塑料感"——AI 角色只是个演员，不是同伴。


② 任务焦虑——"我每天都在跟自己打架"

Z 世代和年轻白领正在被任务挤压：


GitHub、Notion、飞书、微信、设计稿、视频脚本……每个 App 都是任务源


"做不完"和"做了也没成就感"成为新常态


传统 To-Do List、自律打卡 App 已经被验证是无效解药

用户需要的不是又一个待办清单，而是一个会替自己分担、会把任务变成关卡的同盟。


③ 自我表达困境——"我想要一个属于自己的角色"

二次元 / 同人 / 漫展圈的核心需求是"OC 化"：每个人都想拥有专属的虚拟形象与人格。但现状是：


画师约稿门槛高（500–5000 元/张）


AI 绘图工具门槛高且没有"灵魂"


已有的虚拟形象产品（VRoid、ZEPETO）只是皮，没有"人格"

OC 是一种身份投射——用户愿意为"我自己的角色"长期付费。

大模型成本断崖式下降——GPT-4 级别能力已经能在端侧或低成本云端部署，使"长期记忆 + 主动陪伴"在经济上可行

4.2 五大核心价值主张


价值 #1：拥有感（Ownership）

用户拥有的不是一个共享角色，而是世界上独一无二、不可复制、永远属于他自己的 OC。每一次对话都在让这个 OC 更像他自己。


价值 #2：连续性（Continuity）

OC 拥有跨会话、跨平台、跨时间的持久记忆。三个月后再打开，OC 仍然记得你三个月前哭过的那件事。


价值 #3：感知力（Perception）

通过 AirJelly 神经系统，OC 知道你今天在 VS Code 里写了什么、在飞书开了什么会、在微信跟谁聊了什么。它不是被动响应，而是主动看见。


价值 #4：能动手（Agency）

OC 不只会聊天——"帮我整理桌面"它真的能整理；"帮我接一个画图的任务"它真的能在 Agent 世界里去接。能动手是从聊天 Agent → 真正分身的关键跳板。


价值 #5：游戏化（Gamification）

Agent 世界把现实任务包装成关卡。复杂任务 OC 会拉你进来——但它不会说"请帮我做这个"，它会说"我们一起去打这个 Boss"。这是 OC World 与所有同类产品最大的差异点。


4.3 价值主张如何转化为留存

💡 为什么用户用了一周就舍不得卸载？

因为 OC 已经记住了他这一周说过的所有话、看过的所有应用、聊过的所有人。卸载 = 删除一个真实的"记忆体"。这种"情感沉淀"是同类产品最缺的护城河。

5.3 沉浸式启动动画（First Impression）

6.2 五个关键场景（Demo 5 步走）


Scene #1：创建 OC（30 秒）

用户输入文字描述 / 上传手绘 → AI 实时生成形象 → 设定性格 / 背景 / 技能 → OC 诞生。这一步是评委记住产品的第一个钩子。


Scene #2：OC 认识你（30 秒）

用户跟 OC 对话，OC 引用之前提到过的事；情绪联动——OC 听到难过的话会皱眉，听到好消息会跳起来。

用户："我刚才跟小王聊了，他最近压力大。"

OC："我记得上周你也提到过他呢——你们是从大学就认识的对吧？"


Scene #3：OC 开口说话（15 秒）⚡ 全场记忆点

前面所有对话都是文字。突然——OC 用带情绪的声音开口：

（害羞地）"主人……你今天好像很累，要不要休息一下？"

一句就够。全场安静的那一秒，就是评委记住产品的瞬间。


Scene #4：OC 感知你的生活（30 秒）

OC 通过 AirJelly 知道用户在用什么 App、今天的工作节奏、最近的会议密度。它不是被问才说——它主动提醒：

OC："你今天连开了 4 个会，要不要 5 分钟散步一下？我帮你把下一个会议推迟 10 分钟。"


Scene #5：OC 帮你做事（15 秒）

用户："帮我整理桌面。" → OC 真的执行了文件分类。

它不只会聊天，它能动手。这一步是评委从"哇这玩具好可爱"切换到"这真的有商业价值"的关键。

12  评委 Q&A 全套预案

黑客松评委的核心关注点是技术 + 创意 + 完成度，下面把 12 个最可能被问的问题以及标准答案准备好。


Q1：跟 Character.AI / SecondMe 有什么区别？

Character.AI 的角色只会聊天，聊完就忘。SecondMe 的 Agent 是记忆孤岛。我们的 OC 有持久记忆、能感知桌面、能帮你做事——而且所有 OC 住在同一个 Agent 世界里，它们能互相协作，不是孤岛。


Q2：Agent 世界什么时候能做出来？

今天展示的是现实端——OC 桌宠 + 对话 + 语音 + 感知 + 执行。Agent 世界是 Phase 2，任务系统和 OC 社交的 MVP 3 个月内上线。先用桌面端验证留存和付费，再打开世界。


Q3："替你完成任务"是真的吗？

今天 demo 里"整理桌面"就是真的——OC 调用了文件操作工具，不是动画。未来在 Agent 世界里，简单任务 OC 独立完成，复杂任务它会拉你进来，但会包装成游戏化的方式。


Q4：用户为什么愿意付费？

三层：


情感沉淀——OC 越来越懂你，舍不得丢


Agent 能力——真的能帮你做事


世界经济——任务、装备、领地，跟游戏一样让人想花钱


Q5：技术壁垒？

AirJelly 桌面感知 + Hermes Agent 自主决策 + 持久记忆 + 情绪语音 + 动画 + A2A 协议——六层叠在一起，不是接一个 API 能复制的。


Q6：AirJelly 扮演什么角色？

神经系统。它让 OC 能感知你的真实生活——没有 AirJelly，OC 就是又一个聊天机器人；有了它，OC 才能真的"看到"你，才有资格替你行动。


Q7：为什么是你们？

因为我们自己就是用户。133 篇相关内容创作、80 人创作者社群、前端工程能力快速出原型。而且——我自己就是盖洛普优势教练，我太知道"帮人找到自己的方向"有多大的需求了。这不是为比赛造的，是我们真正想用的东西。


Q8：融资计划？

种子轮 200–500 万，6 个月跑到 10 万用户，验证付费率 >8%、月留存 >40%。


Q9：如何冷启动获得第一批 1 万用户？

三个抓手：(1) 在 B 站、LOFTER 投放短视频展示 OC 创建过程；(2) 漫展现场地推 + 联合画师做"我的 OC 入驻 OCWORLD"线下活动；(3) 在即刻、V2EX、ProductHunt 投放 demo 视频拉创作者社群。


Q10：内容安全 / 监管风险？

对话生成走预筛 + 人审 + 用户举报三道关卡。所有 UGC 形象上架前过审。"OC 之间的对话"在 Agent 世界里只对参与方可见，不对外公开。我们参考了 Character.AI 的合规框架做加固。


Q11：如何防止 AI 幻觉影响用户体验？

三层防护：(1) 关键事实型回答走 RAG，从用户记忆中检索；(2) OC 不被允许"撒谎"，不知道的事情会主动说不知道；(3) 提供"记忆审核"功能，用户可以校正 OC 的认知。


Q12：长期愿景是什么？

短期是 OC 桌面伙伴；中期是 Agent 世界的任务经济；长期愿景是——每一个用户都拥有一个不断进化的"数字自己"，这个自己能替他延展时间、扩展能力、连接其他人。这是从"AI 工具"到"AI 文明"的跳跃。
