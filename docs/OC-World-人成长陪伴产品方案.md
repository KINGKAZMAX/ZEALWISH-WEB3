# 第1章 产品定义

## 1.1 产品是什么

OC World 不是一个普通聊天工具，也不是一个帮用户自动完成一切的 AI 助手。

它要做的是：

**让 OC 成为一个长期成长陪伴体，陪用户观察生活、整理经验、解释卡点、记录努力、提炼能力变化，并把真实生活一步步游戏化。**

这里的游戏化，不是做表层勋章和打卡皮肤，而是给人的成长过程建立一套持续反馈系统，让人能看见自己正在变强。

## 1.2 产品不是什么

OC World 不该被做成以下几类产品：

1. **纯聊天陪伴产品**  
   只有情绪价值，没有成长主线。

2. **纯 Habit Tracker / 人生仪表盘**  
   只有目标、打卡、进度条，没有关系和理解深度。

3. **AI 人生代理**  
   把人的行动、判断和责任都外包给 AI。

产品边界必须清楚：

- 人负责行动
- OC 负责陪伴、记录、反馈、追问、支持
- Agent 能力只是辅助层，不是产品终点

## 1.3 产品服务对象

产品最终服务的是：

**想认真活、想持续变强、但经常会反复、卡住、怀疑自己的人。**

这类人不缺道理，也不缺工具，缺的是：

- 有人持续看见自己
- 有人帮自己把混乱变清楚
- 有人提醒自己不是没进步，只是没被记录
- 有一套机制能把努力转成成长感

# 第2章 产品理念

## 2.1 人是主角，OC 是陪练

OC World 的核心前提是：

**人不是被 AI 替代的对象，人是被 OC 陪着变强的对象。**

OC 的角色更像：

- 成长关系里的镜子
- 行动卡点时的陪练
- 低谷时的支持者
- 长期经验的记录者
- 能力变化的见证者

关系很重要，但关系不是终点。关系存在的意义，是让 OC 越来越懂用户，越来越知道怎样支持用户成长。

## 2.2 记录的不是任务，是成长证据

用户每天做了什么，不应该只被写成 todo done。

产品真正要记录的是：

- 今天练习了什么能力
- 哪个行为说明更稳定了
- 哪个选择说明判断力提高了
- 哪个困难开始愿意面对了
- 哪些卡点仍在反复出现

也就是说，任务只是表层，**成长证据**才是核心记录对象。

## 2.3 游戏化服务真实成长

产品可以有能力值、时间线、阶段、升级感，但这些东西必须有真实依据。

游戏化在这里的作用是三件事：

1. 让成长变得可见
2. 让坚持变得更有反馈
3. 让长期复利变得更容易感知

产品不能鼓励用户沉迷数值本身，而要鼓励用户看见：

**原来自己真的比上周更稳、更清楚、更敢行动。**

## 2.4 OC 和人一起成长

这不是“养一个 AI 宠物”的逻辑。

更准确的关系是：

**人和 OC 在长期相处中互相理解，一起升级。**

用户在成长：

- 更理解自己
- 更能行动
- 更能复盘
- 更有能力面对真实生活

OC 也在成长：

- 更懂用户的情绪模式
- 更懂用户的长期目标
- 更会提问
- 更会支持
- 更能形成稳定陪伴风格

# 第3章 目标用户

## 3.1 核心用户画像

第一阶段最适合的用户，是 20-35 岁的上升期青年。

这类用户的共性：

- 希望自己变强
- 正在工作、学习、创作或转型
- 日常会自我要求，也会自我怀疑
- 经常有很多想法，但执行节奏不稳定
- 愿意接受长期陪伴型产品
- 不满足于冰冷的效率工具

## 3.2 典型痛点

### 痛点一：努力很多，但没有成长感

用户并不是没努力，而是努力分散、没有被记录，也没有被解释成能力变化。

### 痛点二：知道该做什么，但总在关键处卡住

用户缺的不是建议本身，而是一个真正理解自己状态的人，能在关键时刻给出更贴身的支持。

### 痛点三：容易只看到自己没做好的部分

用户的自我反馈系统常常失真，只记得失败，不记得微小进步。

### 痛点四：缺少长期成长关系

很多工具只在“今天要做什么”这一层停住，没有形成“这个人正在变成谁”的长期陪伴关系。

## 3.3 为什么这类用户会长期使用

因为产品提供的不是一次性价值，而是长期关系价值：

- 用得越久，OC 越懂自己
- 被记录得越久，成长越可见
- 复盘越持续，行动能力越强
- 关系越深，支持越有效

# 第4章 用户价值

## 4.1 看见自己

产品把用户散落在一天里的生活信号重新收拢，让用户知道：

- 今天到底发生了什么
- 精力流向了哪里
- 哪些事在消耗自己
- 哪些事在推动自己成长

## 4.2 被理解

OC 不是统一模板回复，而是基于：

- 最近对话
- 长期记忆
- 关系阶段
- 当日使用 App / 任务 / 事件

给出更贴身的回应。

## 4.3 形成行动

产品的目标不是“聊得好”，而是“聊完之后更知道下一步做什么”。

每次对话都应该尽量收束成一个现实可执行的小步，而不是停在安慰或概念层。

## 4.4 看见能力变化

用户真正需要的不是更多提醒，而是看到自己在这些维度上的变化：

- 表达力
- 行动力
- 判断力
- 情绪稳定
- 专注力
- 创造力
- 关系经营
- 自我理解
- 身体能量

这些能力值不是靠手动打卡上涨，而是由真实行为、复盘、上下文、长期记录共同判断出来。

# 第5章 核心产品主循环

OC World 的核心循环应该固定为：

**生活发生 → OC 记录 → 一起复盘 → 提炼能力成长 → 给出下一步练习 → 回到真实生活**

## 5.1 生活发生

现实生活里的事件先发生。

来源包括：

- 用户主动输入
- AirJelly 感知到的事件
- App 使用情况
- 当日任务进展
- 最近社交与记忆素材

## 5.2 OC 记录

OC 把这些信号组织成“今天发生了什么”的结构化理解。

记录的不是流水账，而是：

- 事实
- 情绪
- 卡点
-投入
- 值得记住的变化

## 5.3 一起复盘

OC 通过追问、共情、解释，帮助用户把经历转成理解。

关键不是替用户下结论，而是帮用户更快看见：

- 为什么会卡住
- 哪一步做对了
- 哪个模式又出现了
- 今天真正值得保留的是什么

## 5.4 提炼能力成长

复盘结束后，系统要能提炼出成长事件。

例如：

- 今天顶住压力把话说清楚了 → 表达力 + 判断力
- 今天明明很累但还是推进了关键一步 → 行动力 + 情绪稳定
- 今天发现自己一直回避某类问题 → 自我理解提升

## 5.5 给出下一步练习

成长必须回到现实行动。

所以每一轮高质量互动，都应该尽量长出一个：

- 很小
- 很具体
- 当下可做
- 与用户状态贴合

的下一步练习。

## 5.6 回到真实生活

产品的价值最终不发生在应用内，而发生在真实生活里。

用户带着新的理解和下一步行动离开应用，第二天再带着新的生活证据回来。

# 第6章 功能全景

## 6.1 OC 陪伴对话

这是当前产品的主入口，也是长期最核心的体验层。

能力要求：

- 读懂当日状态
- 结合长期关系说话
- 既能共情，也能追问
- 不停留在陪聊，要能把对话引向理解和行动

现有承接：

- `chat-engine.ts`
- `prompt-builder.ts`
- `useChat.ts`
- `HomeView.tsx`

## 6.2 成长记录

系统要把聊天、上下文、任务、用户输入，沉淀成成长记录。

成长记录至少包含：

- 发生了什么
- 用户情绪状态
- 反映了什么成长信号
- 关联了哪些能力维度
- 这件事为什么值得记住

现有承接基础：

- `growthEvent`
- `relationship.keyMoments`
- `timeline`

## 6.3 能力值系统

能力值是 Growth 模块的核心对象。

建议第一版先做 8-9 个维度：

- 表达力
- 行动力
- 判断力
- 专注力
- 情绪稳定
- 关系经营
- 创造力
- 自我理解
- 身体能量

原则：

- 不能让用户随便自填加分
- 必须来自真实行为证据
- 必须允许波动，不是假装永远上涨
- 要有“为什么这项能力变化了”的解释

## 6.4 成长回溯

当前 `Rewind / Timeline` 更偏关系时间线。

下一步应升级成双层回溯：

1. **关系回溯**：你和 OC 的共同成长
2. **人生回溯**：你真实生活中的成长证据

用户要能回看：

- 什么时候开始变稳
- 哪类问题一直反复
- 哪个阶段进步最快
- 最近能力变化和生活事件之间的关系

## 6.5 每日 / 每周复盘

这是让产品真正形成长期复利的关键模块。

### 每日复盘

目标：

- 收住一天
- 看见真实发生
- 识别一个成长点
- 给出明天的一小步

### 每周复盘

目标：

- 总结连续变化
- 识别反复模式
- 看到能力维度波动
- 帮用户形成更高一级的判断

## 6.6 下一步练习

这是连接“理解”和“改变”的桥。

练习的形式可以是：

- 一句该说出口的话
- 一个只需 10 分钟的动作
- 一次刻意练习
- 一个观察任务
- 一个复盘问题

产品要求：

- 练习要短
- 练习要贴身
- 练习要可验证
- 练习要能回收结果

## 6.7 轻量游戏化系统

游戏化不做重度任务系统，先做轻量反馈层：

- 能力值变化
- 阶段成长称号
- 成长里程碑
- 连续复盘记录
- 特定能力的高光时刻

这些机制用来增强成长感，不是制造打卡焦虑。

## 6.8 OC 共同成长

除了用户成长，OC 也要让人感受到“更懂自己了”。

可体现为：

- 说话越来越贴身
- 提问越来越准确
- 记住更多长期模式
- 在关键时刻更知道怎么支持

这会强化留存，因为用户不是在用工具，而是在进入一段长期成长关系。

# 第7章 用户怎么使用

## 7.1 初次进入

用户首次打开产品，先创建自己的 OC。

这一步的关键不是捏脸，而是建立一种关系预期：

- TA 是谁
- TA 的性格是什么
- TA 会怎样陪自己
- 这段关系是长期的

现有承接：`CreateView.tsx`

## 7.2 日常使用

用户在白天或当下卡住时，直接进入首页或聊天页。

典型动作：

- 说今天发生了什么
- 说自己在烦什么
- 问现在最该先做什么
- 让 OC 帮忙看清状态

现有承接：`HomeView.tsx` 的 Quick Start 和输入主链。

## 7.3 晚间复盘

每天最关键的一次使用，应该是晚间的 5-10 分钟复盘。

流程建议：

1. OC 读取今天的生活信号
2. 用户补充关键经历
3. 一起整理今天的重要变化
4. 提炼一个成长点
5. 给出明天一小步

这是未来增长和留存的核心动作。

## 7.4 周期性回看

用户每周或双周进入回溯与成长页，去看：

- 最近经历了什么
- 哪些问题一直反复
- 哪个能力维度正在提升
- 哪个阶段需要刻意练习

这一步决定产品能不能从日用工具变成长期成长系统。

## 7.5 长期留存机制

长期留存不靠提醒轰炸，而靠三件事：

1. 越用越懂
2. 越记越准
3. 越看越能发现自己真的在变

# 第12章 成功指标

## 12.1 行为指标

- 周活跃用户数
- 人均每周复盘次数
- 含成长记录的对话占比
- 下一步练习的执行回报率

## 12.2 成长指标

- 人均每周新增成长记录数
- 能力维度被触发的丰富度
- 用户连续两周以上出现的成长趋势

## 12.3 体验指标

- 用户主观“被理解感”
- 用户主观“我真的在变好”的感受
- 用户是否愿意长期把真实生活交给 OC 记录和陪伴

## 12.4 关系指标

- 关系深度增长与复盘留存的相关性
