# ZEALWISH · 活世界 — 世界观设定集（Worldview Bible）

> 一句话:**一座由你拥有的活世界。你的 AI 角色在这里自己过日子,你可以随时来串门、化身任何一个 TA、看它们的人生慢慢展开。**

口号:*A companion becomes real when it remembers* → **…when it lives.**

---

## 1. 世界设定

**活世界(Living World)** 是 ZEALWISH 工作台里的一层——你在工作台「创建 / 对话 / 记忆 / 护照」养成一个属于你、归属你钱包的 AI 角色;而在「活世界」里,这个角色不再只是对话框里的影子,而是真正**住在一张可探索的像素世界地图上**,自己走动、相遇、说话、把今天记进心里。

- **基调**:温暖、治愈、慢生活;宝可梦火红(GBA JRPG)风的 2D 俯视像素世界。
- **据点**:角色们聚居在一处**「活动区域」**——默认是东海岸的**海景小屋绿地小岬**(四面环海、木码头、花园)。活动区域可自定义:换预设(海边小屋 / 林海码头 / 海湾绿岛 / 红顶小镇 / 林间空地 / 金桥湖畔 / 南滨沙滩 / 群岛浅滩),或「设此为区」把任意地点设为聚居中心。群岛浅滩是一片可涉水的岛礁水域,玩法观感与陆地聚落完全不同;雷达小地图会标出所有区域预设的方位(范围外贴边指向),整张大地图可导航。
- **时间**:世界有自己的昼夜与「纪元(EPOCH)」节律;角色昼夜作息、心情、经济(供给 ◈)随之流转。
- **所有制**:每个角色有自己的身份与钱包(护照);角色属于**你**,不困在平台数据库里。

## 2. 主角 · 小智(你的 OC)

- 钱包拥有的主权 AI 角色,世界的「你」。沿用宝可梦主角(Red)造型。
- 性格:**话少、内敛、字短**,但都记得。「…嗯。」「看着你们,挺好。」「别怕,有我。」
- 你可以亲自操控小智自由行走,也可以接管世界里任何一个居民,体验「成为别人」。

## 4. 核心循环(世界如何「活」)

1. **自治生活**:居民按作息在活动区域里走动、上班、恰饭、发呆。
2. **亲密相会**:每隔一会儿两位伙伴会自己凑到一起,说成对的悄悄话、头顶冒 ♥,关系(好感)随相处增长。
3. **实时社交流(THE FEED)**:他们的生活被写成一条条动态;点动态,镜头会飞向当事人。
4. **你的介入**:点空地→走过去;点身边的 TA→互动(闲聊/夸夸/约饭/抱抱/陪走);点远处居民→接管化身;走近按空格也可互动。互动会累积好感、冒表情。
5. **沉淀**:经历进入角色记忆;在工作台里它会主动引用今天的亲历(记忆驱动对话)。
6. **灵宠**:每个角色有自己的背包与队伍,默认随身宠是圆滚滚的原创萌宠「泡芙」,可随行(可隐藏)、可在活动区域走近野生灵宠按 C 收服(耗灵石)、用能量果喂养涨羁绊与等级。**不同活动区域出没不同的野生物种组合**(由区域坐标确定),换区探索能遇到不一样的灵宠。玩家本地态,不进确定性世界引擎、不参与联机主机权威,纯粹是「你和你的角色之间」的玩法层。

## 5. 设计原则

- **拥有感**:角色与记忆属于用户,可导出、可跨世界携带(护照)。
- **温柔不打扰**:UI 极简、无遮挡控件;世界自己运转,你来不来它都在生活。
- **一体化**:工作台(创建/对话/记忆/护照)与活世界是同一外壳的两面,相辅相成。
- **小智永不消失**:小智是活世界的永久主角。即便玩家创建并操控了自创角色,小智仍会作为常驻居民留在世界里(可一键「回到小智」),绝不因为创建新角色而消失。
- **版权状态(如实说明)**:世界地图与角色精灵**图形本体**取自 Pokémon FireRed/LeafGreen(取自反汇编工程 `pret/pokefirered`),以「非商用粉丝/学习项目」声明使用,版权归任天堂/Game Freak/Creatures Inc.,详见 `living-oc/public/sprites/ASSETS-NOTICE.md`。除此之外——灵宠系统(物种/道具/美术)、HUD/对话框的 GBA 框风皮肤(米白框 + 深蓝描边,原创复刻而非搬运素材)、字体接入管线、生命引擎、对话、经济、整套观测台逻辑均为本项目自研,与上述素材无关。

## 6. 访客如何进入探索

目标:让**别的用户**也能进来逛你的活世界——轻量上手、可分享、有边界、不破坏 AI 自治。
机制按易实现度排序(每条都有 2026 已验证先例,见 §7):

1. **【已落地】分享链接 → 只读观光,无需登录**(`?visit=1`)
   - 任何人打开 `…/world/?visit=1` 即进入**访客观光模式**:世界照常自治运转,镜头**自动巡游**(每 ~10s 换一位居民聚焦),访客**点任意 TA** 可聚焦并查看其人格/记忆,点 THE FEED 动态可飞向当事人。
   - **纯只读**:不能操控、不能接管、不能互动、不能改区域——保护 AI 自治(规避 AI Village「开放插话破坏自治」的坑)。
   - 顶部显示「👁 访客观光 · 只读串门」徽章 + 「✦ 创建你的活世界 →」转化入口。
   - owner 端「🔗 分享观光」按钮一键复制只读链接,发给朋友来串门。
2. **【下一步】可分享片段 / 精彩回放**:把"相遇 / 悄悄话 / 关系升温"等涌现瞬间做成可分享卡片或短片(参 Smallville demo 的压缩+可调速渲染)。
3. **【中期】多人在场 presence**:多个访客同时在线、彼此光标/头像可见(可直接用 Liveblocks Presence 或 PartyKit,无需自研后端)。
4. **【谨慎】host 门控的可互动串门**:默认只读,owner 授权后访客才能**低侵入互动**(悄悄话 / 留言 / 送礼),绝不开放自由插话(参 Agentic Island 的 open/restricted 门控模型)。

## 7. 赛道坐标与定位(2026,联网调研)

- **品类母版**:Stanford 生成式智能体(Smallville,UIST'23)—— 25 个自治 AI 在仿《模拟人生》沙盒里生活,从单一意图涌现出社交事件;a16z **AI Town**(MIT 开源、可 fork)是其工程范式;Altera **Project Sid** 把规模推到 500 智能体的"AI 文明"叙事。
- **访客机制先例**:**Agentic Island**(分享链接即看、无需登录、host 经 MCP 门控只读 vs 可互动)、**Smallville/AI Town**(浏览器 2D 实时画布 + replay/demo)、**AI Village**(AI 真人秀直播,踩过"开放观众插话破坏自治"的坑)。
- **2026 定位(避开 metaverse 包袱)**:参 **Status AI**——"不要被动看故事,而是住进/串门进活着的世界",品类用"沉浸式/互动娱乐"语系。
- **ZEALWISH 的差异点**:**钱包拥有(ownership)× 自治生活(aliveness)× 可串门(visitability)**;不同于第一人称扮演(Status),我们主打"**别人来旁观/串门你拥有的、自治生活着的 AI**"。
- **一句话 pitch 候选**:
  - 你拥有的 AI,在自己的世界里真的活着 —— 朋友可以来串门。
  - 不是看 AI,是去 AI 住的地方串门。
  - 一个钱包拥有、自治生活的小世界;分享一个链接,任何人都能来看。
